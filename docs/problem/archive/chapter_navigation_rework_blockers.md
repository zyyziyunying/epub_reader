## 阻塞问题

### 1. 当前状态描述与代码现状不一致

状态：已解决（2026-03-20）

`chapter_navigation_rework.md` 把一部分“历史问题回顾”和“当前待执行步骤”混写在一起，导致执行状态不清晰。

当前代码已经处于部分 `Phase 0` 之后的形态：

- 阅读页当前是普通连续滚动渲染
- 目录抽屉中的章节项不可点击
- 底部栏已明确提示章节导航暂时禁用

在状态没有重写清楚前，不能把该文档当成可靠实施依据。

本项已通过以下调整解决：

- 在主文档中新增“文档定位”
- 在主文档中新增“当前代码状态”
- 将旧实现问题与当前代码状态分离
- 将 Phase 0 改写为已落地的清理结果，而不是待执行步骤

### 2. Phase 1 范围存在自相矛盾

状态：已解决（2026-03-20）

文档一方面要求：

- 用 EPUB TOC 重建目录
- 上一章/下一章按目录项移动

另一方面又明确不包含：

- `href#anchor` 精确跳转
- 同文件多个目录点定位

这会导致多个目录项可能映射到同一个正文文档顶部，出现“目录项不同，但跳转结果相同”的不一致行为。

执行前必须先明确：

- Phase 1 对这类 TOC 节点的退化策略
- UI 是否允许点击但显示为近似跳转
- 上一章/下一章到底按 `TocItem` 还是按 `ReaderDocument` 移动

本项已通过以下调整解决：

- 在主文档中新增“Phase 1 范围收敛”
- 明确 Phase 1 的唯一直接定位单元是 `ReaderDocument`
- 明确 Phase 1 目录 UI 只消费派生的 `DocumentNavItem[]`
- 明确细粒度 TOC 节点延后到 Phase 2，而不是伪装成精确跳转
- 明确上一章/下一章在 Phase 1 中按 `ReaderDocument` 移动

### 3. 缺少 TOC 到正文的稳定映射契约

状态：已解决（2026-03-20）

当前问题文档提出了 `ReaderDocument` 和 `TocItem` 两层模型，但没有定义最关键的映射规则：

- `href` 如何规范化
- 相对路径如何解析
- URL encode/decode 如何处理
- nav 与 NCX 冲突时如何取舍
- TOC 顺序与 spine 顺序不一致时如何处理
- `targetDocumentIndex` 的生成规则是什么

如果这些规则不先写清楚，后续实现会在解析层和阅读器层反复返工。

本项已通过以下调整解决：

- 在主文档中新增“TOC 与正文映射契约”
- 明确 `ReaderDocument` 与 `TocItem` 的来源和职责边界
- 明确 `href` 规范化、fragment 拆分、路径归一化和精确匹配规则
- 明确 unresolved `TocItem` 的处理方式
- 明确文档级导航项的派生规则和顺序规则

### 4. 进度模型字段定义仍然不足

状态：已解决（2026-03-20）

文档把进度从“全书滚动百分比”调整为：

- `documentIndex`
- `documentProgress`
- `tocItemId` 可选
- `anchor` 可选

但 `documentProgress` 目前仍然没有精确定义。

执行前必须明确它至少包括以下内容之一：

- 文档内滚动百分比的精确定义及误差接受范围
- 基于锚点或结构节点的恢复策略
- 图片异步加载、字体变更、样式变更后的恢复退化行为

否则只是把误差从整本书缩小到单文档，并没有真正解决恢复精度问题。

本项已通过以下调整解决：

- 在主文档中把“进度模型方向”改为“进度模型定义”
- 明确 `documentProgress` 的范围和计算语义
- 明确恢复优先级和 Phase 1 的误差边界
- 明确字体、样式、图片变化后的退化行为只允许停留在同一 `ReaderDocument`

### 5. 数据迁移方案缺失

状态：已解决（2026-03-20）

当前数据库仍然是旧模型：

- `chapters` 表只保存 `chapter_index/title/content`
- `reading_progress` 表只保存 `chapter_index/scroll_position`
- 数据库版本仍为 `1`
- 没有 `onUpgrade`

这意味着重构不是单纯的模型新增，而是一次明确的数据迁移工作。

执行前必须明确：

- 是新增表、替换表，还是保留旧表并做双轨迁移
- 已导入书籍是自动迁移、懒迁移还是强制重建
- 迁移失败时如何回退
- 旧进度数据如何失效或映射

本项已通过以下调整解决：

- 在主文档中新增“数据迁移策略”
- 明确采用“新增 V2 导航数据 + 旧书按需重建”的路线
- 明确保留旧表作为过渡期遗留数据，而不是原地硬改
- 明确旧书重建失败时维持 Phase 0 最小阅读体验
- 明确旧进度只做 best-effort 映射，不承诺无损迁移

### 6. 缺少验收标准和测试矩阵

状态：已解决（2026-03-20）

当前文档没有定义“做到什么程度算 Phase 1 完成”，也没有列出必须覆盖的 EPUB 样本。

执行前必须补齐至少以下内容：

- 最小可接受功能清单
- 必测 EPUB 场景清单
- 回归测试范围
- 不支持场景的显式声明

建议最少覆盖：

- TOC 顺序与正文顺序一致的普通 EPUB
- TOC 指向 `href#anchor` 的 EPUB
- 同一 XHTML 文件含多个 TOC 节点的 EPUB
- 无标题或标题质量较差的正文文件
- 图片较多、文档长短差异明显的内容

本项已通过以下调整解决：

- 在主文档中新增“验收标准与测试矩阵”
- 明确 Phase 1 完成标准
- 明确必测 EPUB 场景和回归范围
- 明确 Phase 1 的不支持边界，避免实现时暗含额外承诺
